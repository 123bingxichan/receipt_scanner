# Contributing

## Local Workflow

1. Create a branch for the change.
2. Keep changes focused.
3. Run formatting, type checks, linting, and tests before opening a pull request.
4. Update docs when behavior, setup, or architecture changes.

## Commit Style

Use short, imperative commit messages:

- `Add receipt upload page`
- `Implement OCR adapter contract`
- `Fix expense category filtering`
